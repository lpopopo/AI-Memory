## 一、问题诊断摘要（2026-06-22 至 2026-06-27 实盘复盘）

### P1 — 主题集中度失控（最严重）

**现象：**  
5个持仓（GLW / TTMI / MXL / DRAM / MU）全属同一 AI Capex 链。  
6/23 半导体暴跌日，5仓全线同向下跌，无对冲。

**数据佐证：**
- GLW（光通信）-10.92%
- TTMI（PCB/互联）-5.39%
- DRAM（存储ETF）-12.71%
- MU（HBM存储）-6.70%（财报后次日）
- MXL（光学DSP）同向

**根因：**  
6/25 单日连续开仓 DRAM → MXL → MU，三次操作全是同一主题方向，但每次分别通过了「单笔」检查，绕过了「主题整体」检查。

**已知规则漏洞：**  
当前 `portfolio-concentration-rules.md` 只限制单股权重，未限制「同日同主题累计新增」上限。

---

### P2 — 单日加仓速度过快

**现象：**  
2026-06-25：组合总敞口从 ~21.58% 跳升至 ~47.24%（单日增加约 +25.66pp）

**规则上限（既有）：** 先 30-35% → 确认后 45-60%  
**实际执行：** 跳过第一阶段直接到第二阶段终点

**后果：**  
6/26 次日三个新仓同时进入 near-stop 区域：
- DRAM $71.88（距止损线 $70.50 仅 +1.96%）
- MU $1132（距止损线 $1100 仅 +2.94%）
- TTMI $191.49（距止损线 $188 仅 +1.86%）

---

### P3 — 财报爆发日追涨（MU）

**现象：**  
MU 财报超预期后 6/25 单日 +15.7%，在当日高点区间买入 1 股 @$1155。

**次日结果：** -6.70%，立即进入 near-stop 区域（成本价-2%）

**违反的隐含规则：**  
「重大财报盘后跳涨后需等待价格稳定再买入」  
该规则当时未明文写入 decisions.md，属于规则空白。

---

### P4 — 动态止损机制缺失

**现象：**  
GLW 成本 $181.50，最高冲至 $228（+25.7% 浮盈），但利润保护线仍设在 $210（仅锁定 +15.7%）。  
6/26 收盘 $221.05，距 $210 线已有 +5.26% 的余地，但没有追踪上移。

**后果：**  
若 GLW 从 $228 直线跌回 $210，会损失 8pp 的账面利润，仅锁定原始+15.7%。

---

### P5 — 卫星仓规模超标（MXL）

**现象：**  
MXL 买入 6 股 @$90.70 = $544.2 = NAV **9.08%**  
卫星仓规则上限：3-6%  
实际占用：**超出上限 50%-200%**

**后果：**  
止损设置偏宽松（$86，卫星仓标准），但实质上承担了核心仓规模，风险敞口不匹配。

## 三、新稳定规则内容（Task-002 写入 decisions.md）

以下6条规则将在 `2026-06-27` 节点追加到 decisions.md：

### 规则 A：单日加仓速度上限
```
在 elevated 恐惧门控状态下，单一交易日内所有新买入操作合计
不得使总组合敞口增加超过 8 个百分点。
在 normal 恐惧门控状态下，单日上限为 15 个百分点。
在 stress 或 panic 状态下，禁止任何新买入。
该规则按已确认成交的真实仓位计算，不含挂单。
```

### 规则 B：同日同主题新增上限
```
同一大主题（如 AI Capex 链：存储/光通信/PCB互联/计算）下，
单一交易日内新增仓位合计不得超过 NAV 的 15%。
本规则叠加在规则A之上，取更严格的一方约束。
```

### 规则 C：财报爆发日冷静期
```
当某股票因财报/重大利好单日涨幅超过 10%，
禁止在该事件日当天或次日（T+1）买入该股票或同主题相关ETF。
最早可于 T+2 评估买入，且需满足：
1. 价格已在高点下方稳定（日内波动 < 3%）
2. 成交量恢复正常水平（<事件日的50%）
3. 恐惧门控未变差
```

### 规则 D：盈利追踪止损机制
```
核心仓浮盈达到 +15%：将完成收盘止损线上移至「成本价 + 8%」（锁定8%利润）
核心仓浮盈达到 +25%：止损线上移至「成本价 + 15%」（锁定15%利润）
核心仓浮盈达到 +40%：止损线上移至「成本价 + 25%」（锁定25%利润）
卫星仓：从最高收盘价起 25% 追踪止损（与 Double-Radar 规则一致）
追踪止损线只能上移，不得下移。
每次正式盘后审计必须检查是否需要更新各仓位的追踪止损线。
```

### 规则 E：主题浓度硬上限
```
单一子主题（如"AI HBM存储"）总仓位不得超过 NAV 25%。
单一大主题（如"AI Capex"）下所有子主题合计不得超过 NAV 55%。
若同一大主题下仓位已达 40%，
则任何来自该主题的新买入申请必须同时减仓等量的同主题既有仓位。
以上以真实确认持仓计算，不含挂单。
```

### 规则 F：卫星仓规模定义与执行
```
卫星仓：单名仓位目标权重 3%-6%（HKD 50,000账户对应 $200-$400 市值）
  → 对应约 2-4股（$90-$100 价位股票）或 1股（$300-$400 价位股票）
核心仓：单名仓位目标权重 8%-15%
  → 对应约 5-8股（$90价位）或 1-2股（$500-$750价位）
若某仓位当前市值已超出其定义分类上限，
应在下次正式审计时重新定性并调整止损标准。
MXL 6股 @$90.70 即时重新定性为「核心仓」，
止损线从卫星标准 $86 上移至核心标准 $91-92。
```
